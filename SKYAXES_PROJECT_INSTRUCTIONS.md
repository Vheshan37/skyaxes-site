## 1. Core Positioning (USE CONSISTENTLY)

> **"Japan-side PM-led offshore development partner for 3D/VR, IoT and business systems in manufacturing and dental/medical-adjacent technical fields."**

- Use **"Japan-side PM-led offshore development"** as the main phrase everywhere
- Use **"realistic development cost"** / **"cost-effective technical development"** — NOT "cheap"
- SkyAxes role: requirements → specifications → communication → QA → maintenance
- Do NOT position as generic low-cost outsourcing broker

## 2. Brand Rules (MANDATORY)

### Colors — only these, no others
| Token | Hex | Usage |
|---|---|---|
| Navy / Primary | `#1A3C6E` | Headers, nav, footer |
| Accent | `#2E75B6` | CTA buttons, links |
| Surface | `#F0F5FA` | Alternate section backgrounds, cards |
| Success / IoT | `#0D8F6F` | IoT content, success states |
| Clinical | `#4A90C4` | Healthcare/3D pages |
| Text primary | `#2D2D2D` | Body text |
| Text secondary | `#6B7280` | Captions, metadata |

### Fonts — only these two
| Use | Japanese | English |
|---|---|---|
| Headings | Noto Sans JP Bold/Medium | Plus Jakarta Sans Bold/Semibold |
| Body | Noto Sans JP Regular | Plus Jakarta Sans Regular |

### Anti-patterns — NEVER do
- No dark/neon themes (signals gaming, not enterprise)
- No stock photography of Western people
- No auto-playing video
- No excessive animations (subtle hover + scroll-fade only)
- No emojis — use SVG icons only
- No machine-translated content
- No aggressive "cheap" sales copy
- CTAs only at: hero / end of service page / end of case study

## 3. Page-by-Page Instructions

### Home Page — Required sections in order
1. Hero — offshore development as main message
   - Headline: "Japan-side PM-led Offshore Development for 3D/VR, IoT and Business Systems"
   - Sub: "We help manufacturing, dental/medical-adjacent and technical businesses turn specialized software ideas into practical systems at a realistic development cost."
   - Primary CTA: "Discuss an Offshore Development Project"
   - Secondary CTA: "View 3D/VR Case Studies"
2. Three service cards: Offshore Development / 3D·VR·AR / IoT·Manufacturing
3. Featured case study: Dental 3D Viewer
4. Supporting proof: LineQC + AR training (secondary)
5. Development flow: Consultation → Requirements → Estimate → Development → QA → Maintenance
6. Filtered contact CTA (development consultations only)

### Offshore Development Page
- Target: SMEs, manufacturers, dental/medical, equipment makers
- Emphasize Japan-side PM, NOT just a referral agent
- Section order: Who for → Problems → Delivery model → Types → Flow → FAQ → CTA

### 3D/VR/AR Development Page
- Link dental 3D viewer as flagship
- Only technically accurate terms: Qt/OpenGL, 3D viz, data import, Windows app
- Tone: practical, business-oriented — NOT futuristic/entertainment

### Dental 3D Viewer Case Study
| Section | Content |
|---|---|
| Challenge | Dental workflows need specialized 3D display not covered by generic tools |
| Solution | Custom viewer: UI controls, model display, rotation, zoom, extensibility |
| Technology | 3D rendering, Qt/OpenGL, Windows app, software distribution, license control |
| Outcome | Practical cost via Japan-side PM + offshore development |
| CTA | Discuss a similar 3D/VR development project |
> WARNING: Do NOT claim regulatory compliance, diagnostic accuracy or medical certification

### LineQC Page (Section 6.5 from doc)
- Purpose: evidence of manufacturing-system know-how — NOT primary business
- Add "Manufacturing system development proof" summary near top
- Keep detailed features for users already interested
- Add paragraph connecting LineQC to SkyAxes broader factory-system capability
- Use LineQC as proof for manufacturing credibility on home + case study index
- Do NOT let LineQC dominate home page message

## 5. Japanese Web Conventions
- Address: 〒 prefix, Prefecture > City > Ward > Block format
- Phone: 03-XXXX-XXXX domestic, +81-3-XXXX-XXXX international
- Display 法人番号 on Company page (trust signal for gov procurement)
- Reference APPI (個人情報保護法) in privacy policy
- Date: YYYY/MM/DD or YYYY年MM月DD日 — NEVER MM/DD/YYYY
- Language: です/ます form; use お問い合わせ not 連絡
